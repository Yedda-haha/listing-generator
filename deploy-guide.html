部署到 GitHub Pages 指南 — 完全静态，用户数据本地存储

1

创建 GitHub 仓库

登录 GitHub，新建一个仓库：

名称填 listing-generator（或自定义），选择 Public，不要勾选任何初始化选项，然后点 Create repository。

2

推送代码

在本地 listing-generator 目录下执行：

# 初始化 Git 并推送
git init
git add .
git commit -m "init listing generator"
git remote add origin https://github.com/你的用户名/listing-generator.git
git push -u origin main
3

开启 GitHub Pages

进入仓库页面，依次点击：

Settings → 左侧菜单 Pages → Source 下拉框选择 GitHub Actions

注意：Source 一定要选 GitHub Actions，不要选 "Deploy from a branch"，否则部署会失败。
4

等待自动部署

推送代码后 GitHub Actions 会自动触发构建。进入仓库的 Actions 标签页可以查看部署进度。

绿色 ✅ 表示部署成功，整个过程大约 1-3 分钟。

5

访问你的应用

部署成功后，打开以下地址即可使用：

https://你的用户名.github.io/listing-generator/

把这个链接发给别人，他们打开后在页面顶部的 API 配置 里填入自己的 API Key 就能用了。

所有 API Key 存储在用户浏览器本地（localStorage），你这边不保留任何信息。
+

后续更新

修改代码后，只需推送到 main 分支，GitHub Actions 会自动重新部署：

git add .
git commit -m "update"
git push
?

如果用的是项目页（非自定义域名）

如果你的仓库名不是 用户名.github.io，而是类似 listing-generator，需要配置一下路径前缀。

编辑 next.config.ts，加上 basePath：

const nextConfig = {
  output: "export",
  basePath: "/listing-generator",
  images: { unoptimized: true },
};

改完后重新 git push，等 Actions 跑完即可。
